# Port Report: doc-as-oracle harness (rail-redesign)

## build() approach

The new `build()` directly constructs `TabRow { display: TabDisplay { ... } }` without going through the old `StateStore`/`CommandStore`/`aggregate()` machinery. For each tab in the DSL:

1. Each pane line becomes a `PaneDisplay::tracked(pane_id, Kind::from_source(kind), Status::from_wire(status), msg)`.
2. `since_tick` = the pane's 0-based index within its tab's pane list.
3. Tab aggregation mirrors the spec: `total` = pane count, `done` = Done count, `pending` = Pending count. `best_status` walks panes, preferring higher severity; equal severity keeps the later (higher since_tick) pane.
4. Tabs without panes get `TabDisplay { status: Idle, progress: zeros, detail: None, panes: [] }`.

## Baseline: 0 pass / 13 fail

All 13 scenarios fail. The renderer is the old architecture (right-slot elapsed, braille spinner, card tinting, collapse-not-expand). The doc's target design differs on:

### A & B — leading space on tab lines
- **Expected:** `○ 1 shell` (no leading space, glyph at col 0)
- **Got:** ` ○ 1 shell` (1-space indent — the `bar` column for non-active tabs is a space)

### C — no elapsed/spinner in detail line
- **Expected:** `  ✳ running tests…`
- **Got:** `  ✳ running tests… ⠋` (braille spinner appended)
  Also: right slot shows `0:00` elapsed which the doc drops (`⟦D-timer⟧`).

### D — no right-slot on pending
- **Expected:** `◆ 3 api`
- **Got:** ` ◆ 3 api                  ⏵ 0:00` (elapsed + ⏵ in right slot)

### E — done tab still shows detail line; right-slot "done" dropped
- **Expected:** `● 1 dotfiles` + `  ✳ refactored the dotfiles`
- **Got:** ` ● 1 dotfiles               done` (right-slot "done", no detail line for Done status)

### H/I/J/K/M — multi-pane: collapse vs. line-per-pane
The old renderer collapses calm panes into `N working` / `+1 more working`; the new design wants a line per pane. Example K: expected 6 pane lines + `+1 more`; got `7 working`.

### L — overflow strip format
- **Expected:** `+3 idle ▾`
- **Got:** `○ ○ ○ ○ ○ ○ ○ ○ ○ ── +3 idle ▾` (dot leaders prefix)
